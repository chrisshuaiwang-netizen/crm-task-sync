## 1. 产品概述

### 1.1 产品背景

产品经理在日常工作中，频繁接收来自多个客户的需求（微信、会议、邮件），这些需求往往以碎片化的语音或文字形式存在。目前痛点在于：

- 需求记录分散，容易遗漏
- 从"客户诉说"到"可执行任务"缺乏结构化流程
- 工作计划与客户需求之间缺乏双向关联
- 多客户并行管理时，优先级模糊

### 1.2 产品定位

**极简、AI 驱动的个人工作台**

面向产品经理个人使用，帮助其高效将客户需求转化为可执行的工作计划，并实时追踪完成状态。

### 1.3 核心价值主张

| 价值点 | 说明 |
|--------|------|
| 需求捕获 | 随时记录客户诉说（文字/图片/标签） |
| 智能转化 | AI 自动提取需求点并生成任务建议 |
| 双向关联 | 工作计划自动关联所属客户，方便追溯 |
| 状态同步 | 任务完成时，客户需求状态自动更新 |

## 2. 用户角色

## 3. 功能需求详述

### 3.1 仪表盘 (Dashboard)

**功能描述：** 提供全局工作概览，一眼掌握当日待办与重点客户。

**核心组件：**
- 今日任务数量统计（待处理 / 进行中 / 已完成）
- 高优先级客户快速入口
- 近期待处理需求列表（最多 5 条）
- 本周完成进度环形图

**交互逻辑：**
1. 页面加载自动刷新当日数据
2. 点击任意需求卡片，右侧展开详情面板
3. 快速创建按钮（右下角浮动按钮）支持一键新增需求

---

### 3.2 客户管理 (Customers)

**功能描述：** 维护客户档案，关联该客户所有历史需求与当前执行进度。

**数据字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 客户名称 | 文本 | 必填 |
| 行业 | 下拉选择 | 互联网/金融/制造/零售/其他 |
| 优先级 | 枚举 | 高 / 中 / 低 |
| 联系人 | 文本 | 选填 |
| 联系方式 | 文本 | 选填 |
| 备注 | 长文本 | 选填 |
| 创建时间 | 时间戳 | 自动生成 |

**核心功能：**
- 客户列表视图（卡片式，按优先级排序）
- 搜索 + 筛选（按行业、优先级）
- 客户详情页：展示该客户下所有"历史需求"和"当前执行进度"
- 新增 / 编辑 / 归档客户

**交互逻辑：**
1. 点击客户卡片进入详情页
2. 详情页展示需求时间线（按状态分组）
3. 详情页可直接新增该客户的需求

---

### 3.3 需求收件箱 (Inbox)

**功能描述：** 快速记录来自客户的碎片化需求，支持 AI 智能标签与任务生成。

**数据字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 需求标题 | 文本 | 自动提取或手动填写 |
| 需求内容 | 长文本 | 原始描述 |
| 所属客户 | 关联 | 关联 Customers |
| AI 标签 | 多选 | 功能优化/Bug修复/新需求/数据需求/体验优化 |
| 优先级 | 枚举 | 高/中/低，AI 建议 |
| 状态 | 枚举 | 待评审→开发中→已上线→已拒绝 |
| 来源 | 枚举 | 微信/会议/邮件/电话/其他 |
| 创建时间 | 时间戳 | 自动 |
| 截止日期 | 日期 | 选填 |

**AI 智能特性：**
- **智能标签：** 输入需求内容后，AI 自动分析并打上分类标签
- **AI 摘要：** 自动提炼需求的核心诉求（1-2句话）
- **优先级建议：** 根据关键词（如"紧急""ASAP""影响收入"）自动建议优先级
- **任务生成：** 一键让 AI 将需求转化为工作任务（含预估工时）

**状态流转：**
```
待评审 → 开发中 → 已上线
    ↘ 已拒绝
```

**交互逻辑：**
1. 点击"快速记录"，弹出输入框（支持粘贴文本）
2. 输入完成后，AI 自动分析（约 1 秒），展示标签建议
3. PM 确认或修改标签，点击"生成任务"
4. 任务自动进入工作计划（Schedule）

---

### 3.4 工作计划 (Schedule)

**功能描述：** 以日历/看板形式管理每日工作任务，每个任务都关联到具体的客户需求。

**数据字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 任务标题 | 文本 | 必填 |
| 任务描述 | 长文本 | 选填 |
| 关联需求 | 关联 | 关联 Inbox 中的需求 |
| 所属客户 | 自动同步 | 从关联需求获取 |
| 计划日期 | 日期 | 必填 |
| 预计工时 | 数字 | 单位：小时 |
| 状态 | 枚举 | 待处理/进行中/已完成 |
| 优先级 | 枚举 | 高/中/低 |

**三视图切换：**
- **今日任务：** 仅展示今天需要处理的任务
- **本周重点：** 本周所有任务，按优先级分组
- **已完成：** 历史完成记录

**AI 建议功能：**
- 根据截止日期和客户优先级，自动排序推荐当日工作顺序
- 任务超期提醒（高亮展示）

**交互逻辑：**
1. 任务完成时，点击"完成"按钮
2. 系统自动检查关联需求下是否所有任务都已完成
3. 若完成，提示 PM 将需求状态更新为"已上线"
4. 需求状态更新后，客户详情页的进度自动刷新

## 4. 业务流程

### 4.1 核心工作流

```
客户发来需求
    ↓
PM 打开 Inbox → 快速记录原文
    ↓
AI 自动分析 → 建议标签 + 优先级
    ↓
PM 确认 → 点击「生成任务」
    ↓
任务进入 Schedule → PM 排期
    ↓
执行任务 → 标记完成
    ↓
客户需求状态自动更新 → Dashboard 数据刷新
```

### 4.2 数据关联关系

```
Customer (1) ──── (N) Requirement (Inbox)
Requirement (1) ── (N) Task (Schedule)
```
